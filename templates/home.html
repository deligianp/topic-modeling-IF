Topic Model Visualization Application

The web application uses a basic LDA model as the model of reference. The system supports the storage of many LDA models, but only one LDA is configured with the responsibility of performing topic analysis. This policy aims to automatically schedule topic analysis over each stored article and resolve any ambiguity issues that arise while selecting the model that will be used for topic analysis.

The proper functionality of the interface depends on the existence of texts, that have been topically analyzed by the application's main LDA model. These texts, also used in other renown topic model visualization applications, offer valuable insights about the quality of the trained model. Apart from the stored texts, the application can use the LDA model to analyze texts that have not been used during training, thus further expanding the potential of evaluation of the model's quality.
